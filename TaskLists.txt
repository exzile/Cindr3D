SDK CORRECTNESS AUDIT — HIGH SEVERITY
═══════════════════════════════════════════════════════════════
These are correctness bugs: either DesignCAD stores data in a way that can't round-trip
Fusion 360 semantics, or user-visible behavior differs from Fusion. Verified against SDK
headers at C:\Users\joeyp\AppData\Local\Autodesk\webdeploy\production\ca305acf3852cfce8e837ee5435adf649bc398ca\CPP\include\Fusion\.

[ ] CORR-1. Linear dimensions missing orientation type. Fusion SDK
            `SketchLinearDimension.h` + `DimensionOrientations` enum defines
            Aligned / Horizontal / Vertical orientations — these control dimension
            placement rules. DesignCAD's `SketchDimension` (src/types/cad.ts:200-207)
            only stores type='linear'; no orientation discriminator. A horizontal-only
            dimension on an angled edge is currently indistinguishable from aligned.
            FIX: add `orientation?: 'aligned' | 'horizontal' | 'vertical'` to SketchDimension
            and update SketchDimensionPanel + solver to respect it. (medium effort)

[ ] CORR-2. Extrude missing Two-Sides asymmetric extent. SDK
            `ExtrudeFeatureInput.setTwoSidesExtent(distanceOne, distanceTwo, ...)` allows
            different distances on each side of the profile. DesignCAD only has
            'positive' | 'symmetric' | 'negative' — symmetric forces equal both sides.
            FIX: add a `two-sided` direction option with distanceOne/distanceTwo fields.
            Requires GeometryEngine.buildExtrudeFeatureMesh signature change. (medium)

[ ] CORR-3. Fillet missing Variable-Radius and Chord-Length types as proper edge-sets.
            SDK `FilletFeatureInput.h` distinguishes ConstantRadiusEdgeSet,
            VariableRadiusEdgeSet, ChordLengthEdgeSet. DesignCAD's fillet dialog
            has fields for startRadius/endRadius but treats them as a uniform
            variable across all selected edges; each edge can't have its own radius
            function. FIX: change fillet params to accept an array of edge-sets, each
            with its own type. (hard — affects geometry engine)

[ ] CORR-4. Component vs Occurrence distinction missing. SDK `Component.h` (definition)
            and `Occurrence.h` (placed instance) are separate entities — one Component
            can have many Occurrences at different transforms. DesignCAD's
            `Component` type (src/types/cad.ts) is used for both. This means:
            (a) inserting the same component twice copies all bodies instead of
                instancing them.
            (b) changes to an instance can't propagate back to the definition.
            FIX: split into ComponentDefinition + ComponentOccurrence types; store
            transform on the occurrence. (hard — assembly-wide refactor)

[ ] CORR-5. `grounded` flag on wrong entity. SDK `Occurrence.isGrounded()` —
            grounding is per-instance. DesignCAD's `component.grounded`
            (cad.ts ~line 362) is on the definition, so grounding one instance
            incorrectly grounds all copies. Fix is blocked by CORR-4.

SDK CORRECTNESS AUDIT — MEDIUM SEVERITY
═══════════════════════════════════════════════════════════════

[ ] CORR-6. Sketch display flags missing. SDK `Sketch.h` has
            `arePointsShown`, `areProfilesShown`, `areDimensionsShown`,
            `areConstraintsShown` as per-sketch properties. DesignCAD has
            GLOBAL toggles in SketchPalette instead — every sketch shares the
            same display settings. FIX: move these flags to the Sketch type so
            settings persist per sketch.

[ ] CORR-7. `Sketch.isComputeDeferred` missing. SDK flag suppresses automatic
            re-solve while bulk-editing. DesignCAD always re-solves on every
            change — expensive on large sketches. FIX: add a deferred flag
            consulted by solveSketch().

[ ] CORR-8. Extrude `StartExtent` variants incomplete. SDK supports
            `ProfilePlaneStartDefinition`, `OffsetStartDefinition`,
            `EntityStartDefinition` (start from a face/plane/sketch point).
            DesignCAD supports only profile+offset; can't start from another
            entity.

[ ] CORR-9. Extrude `thinExtrudeWallLocation` enum. SDK values are
            `Side1ThinWallLocation`, `Side2ThinWallLocation`,
            `CenteredThinWallLocation` (which we just aligned). BUT SDK allows
            different locations for side1 and side2 independently; DesignCAD
            has a single `thinSide` value covering both. FIX: split into
            thinSideOne / thinSideTwo when two-sided extrude is added (CORR-2).

[ ] CORR-10. Revolve `isProjectAxis` missing. SDK flag controls whether the
             rotation axis is projected to the profile plane or used at its
             original 3D location. DesignCAD always uses the axis as-is.
             Can produce wrong revolve center when axis is skew to the sketch.

[ ] CORR-11. Fillet `isG2` (curvature continuity) not exposed. SDK
             `FilletFeatureInput.isG2` toggles between G1 (tangent) and G2
             (curvature-continuous) fillet blending. DesignCAD's fillet is
             always G1. FIX: add a G1/G2 radio in FilletDialog.

[ ] CORR-12. Fillet `isRollingBallCorner` missing. SDK flag for corner
             setback solution method. Small niche but changes how fillets
             meet at vertices.

[ ] CORR-13. Hole `tipAngle` default not 118°. SDK default is 118° (standard
             twist-drill cone). DesignCAD's HoleDialog uses 90° as a starting
             value in some code paths. FIX: default drillAngle=118 everywhere
             and clamp to [60, 150] as Fusion does.

[ ] CORR-14. Hole `participantBodies` missing. SDK lets the hole feature
             declare which bodies the cut applies to; others pass through.
             DesignCAD cuts all visible bodies. FIX: add an optional
             participantBodies selector; default keeps current behaviour.

[ ] CORR-15. Component missing origin entities. SDK Component exposes
             `originConstructionPoint`, `xConstructionAxis`, `yConstructionAxis`,
             `zConstructionAxis` in addition to the three origin planes.
             DesignCAD only renders an "Origin" folder label (ComponentNode.tsx
             ~line 149-155) — no clickable origin point or axes. FIX: render
             full origin tree (point + 3 axes + 3 planes) and allow selecting
             them as references.

[ ] CORR-16. Construction collections not exposed on Component. SDK Component
             has `constructionPlanes`, `constructionAxes`, `constructionPoints`
             as first-class collections. DesignCAD collapses these into one
             `constructionIds` array — can't query "all construction points in
             this component" cleanly. Cosmetic in UI, but limits future
             reference-selector work.

[ ] CORR-17. Timeline nested groups missing. SDK `TimelineGroup` inherits
             from `TimelineObject`, so groups can contain other groups.
             DesignCAD's `FeatureGroup` is flat — a group can only hold
             features, not other groups. FIX: allow groupId on FeatureGroup,
             render nested indentation in Timeline.tsx.

[ ] CORR-18. No first-class MeshBody type. SDK `MeshBody.h` is a peer of
             `BRepBody` with its own name/visibility/material/appearance.
             DesignCAD treats mesh as a `BodyKind` string on Body; mesh bodies
             can't carry mesh-specific properties (triangle count, repair
             state, etc.) cleanly. FIX: add MeshBody type or mesh-specific
             fields on Body.
