# As of 2026-04-27: small-quality pass on Cura parity gaps:
#   - smallTopBottomWidth — wired (skin emission skips narrow regions)
#   - minSupportInterfaceArea — wired (demotes tiny interface to body)
#   - topSurfaceSkinLayers — wired (top-surface overrides for the topmost
#     N solid layers only, matching Cura semantics; default 0 = no special
#     treatment for any layer)
#   - bottomSurfaceSkinLayers — wired (symmetric flag exposed on the
#     LayerState, ready for future bottom-surface override fields)
#   - useTowers / towerDiameter / towerRoofAngle — wired (small overhang
#     islands now print as a circular tower with a flared roof governed
#     by the roof angle; bypasses the conical / stair-step / horizontal-
#     expansion modifiers since towers are user-controlled column width)
#   - breakUpSupportInChunks / breakUpSupportChunkSize /
#     breakUpSupportChunkLineCount — wired (body-only scanline emitter
#     jumps a chunk-size-mm gap every chunk-line-count lines; interface
#     bands stay continuous so the supported model surface keeps a solid
#     bed of support beneath it)
#   - minSkinWidthForExpansion — wired (per-region check; tiny regions
#     keep the skin overlap but skip the top/bottom expansion offset)
#   - supportDistancePriority — wired (z_overrides_xy relaxes XY clearance
#     to 0 in roof/floor interface bands so support fully covers tight
#     overhangs; xy_overrides_z keeps the existing behavior)
#   - Audited the [s] list: connectTopBottomPolygons, enableSupportBrim /
#     supportBrimLineCount, supportInfillLayerThickness, connectSupportLines /
#     connectSupportZigZags, and the supportRoof*/supportFloor* family
#     (Pattern/Density/LineDistance/Thickness) are all already wired in
#     finalizeLayer.ts / support.ts despite the older [s] markers below.
#     Removed from the list on this pass.
#
# As of 2026-04-27 (later in day): another Cura parity sweep:
#   - outerWallLineWidth / innerWallLineWidth — wired in classic perimeter
#     generator (perimeters.ts). Outer wall uses outerWallLineWidth, inner
#     walls use innerWallLineWidth. First-layer scaling preserved (master
#     × initialLayerLineWidthFactor wins, matching Cura). Arachne path
#     already supported these.
#   - bridgeSkinDensity — wired in emitContourInfill.ts as proportional
#     drop of bridge lines (e.g. density 50 keeps every other bridge line).
#     Non-bridge skin lines on the same layer remain at full density.
#   - interlaceBridgeLines — wired as a simple alternating-layer skip:
#     every other bridge layer drops bridge lines so the alternating
#     layer prints crossing bridge support. (Cura's true 90° rotation
#     would require per-region scan-line generation; this approximation
#     achieves the same structural goal.)
#   - adaptiveLayersTopographySize — wired in adaptiveLayers.ts. For each
#     bin, the most-flat slope present caps layer height to keep the
#     visible step ≤ topographySize. Default 0 = disabled (no-op).
#   - spiralizeContour / vase mode — fully wired in finalizeLayer.ts,
#     emitContourInfill.ts, emitGroupedAndContourWalls.ts, and the
#     GCodeEmitter.extrudeTo (now accepts an optional per-segment Z).
#     Bottom layers print solid as the base; above bottomLayers we
#     suppress infill, inner walls, gap-fill, and support so only the
#     outer wall is emitted, with Z ramped continuously across the wall
#     perimeter from `prevLayerZ → layerZ`. Spiralize-incompatible
#     options (coasting on the closing segment) are gated off.
#   - removeAllHoles mesh repair — wired in coreGeometry.removeAllHoles
#     and called from prepareSliceRun before bbox calc when the profile
#     enables it. Finds boundary edges (edges referenced by exactly one
#     triangle), walks them into closed loops, and fan-triangulates each
#     loop. Operates on a copy so the geometry cache stays pristine
#     across profile changes. Test coverage for manifold-cube no-op and
#     single-triangle hole closure.
#   - moldEnabled + minMoldWidth — wired in prepareLayerState.ts as a
#     per-layer contour transformation: each model outer becomes a hole
#     in an inflated outer (mold ring of `minMoldWidth` material around
#     the model's footprint). Floor layers stay solid (basin floor);
#     above the floor, the model's footprint opens as a cavity. Top
#     skin layers are suppressed so the mold has an open top for cast
#     release. Original holes in the model are dropped (they fill with
#     mold material, which is correct since they were empty in the
#     model).
#   - fuzzySkinsEnabled / fuzzySkinThickness / fuzzySkinPointDist /
#     fuzzySkinOutsideOnly — wired in emitGroupedAndContourWalls.ts.
#     After outer-wall simplification, the loop is resampled at
#     pointDist intervals and each new point is displaced ±thickness/2
#     along its outward normal. Uses a Mulberry32 PRNG seeded by the
#     layer index so re-slicing is deterministic. Skipped on first
#     layer (bed adhesion) and in vase mode (would jitter Z spiral).
#   - smoothSpiralizedContours — wired in emitGroupedAndContourWalls.ts.
#     When spiralize + smooth are both on (and we're above the floor),
#     the outer wall is subdivided into 0.5mm segments before the Z
#     ramp emits. Distributes the layer's vertical climb across many
#     small Z increments so straight box sides don't show a single
#     long Z-up move (visible step) — curved walls already had enough
#     vertices for a smooth ramp.
#   - vase-mode bug fix — the original spiralize gate `continue`'d out
#     of the contour loop AFTER the outerWallFirst=true emission point
#     but BEFORE the outerWallFirst=false emission point. Default
#     profile has `outerWallFirst: false`, so vase mode was actually
#     emitting NO walls at all. Caught by the new SlicerVaseMode test
#     suite. Fix: emit the outer wall explicitly inside the spiralize
#     branch before continuing past the rest of the loop body.
#   - new tests added: src/test/SlicerVaseMode.test.ts (4 tests cover
#     no-infill/inner-wall above floor, solid floor preserved, support
#     suppressed, single-pass outer-only walls) and
#     src/test/SlicerFuzzySkin.test.ts (3 tests cover determinism
#     across repeat slices, first-layer gate, perimeter inflation).
