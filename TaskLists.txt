## Layer-material cache pre-pass  (architectural — unlocks per-region top-skin + multi-layer thickening)
# The current top-skin detection uses a lightweight 1-layer lookahead in
# `emitLayerStartState` (slice + classify + buildMaterial only). The
# current-layer material goes through full prep (modifier meshes,
# horizontal expansion, hole horizontal expansion, mold mode, shrinkage
# compensation, elephant-foot compensation). The boolean
# `currentLayerMaterial − nextLayerMaterial` therefore picks up
# systematic differences from those transforms as spurious top-skin
# regions. Layer-wide solid promotion works around this but is
# approximate. Fixing it requires a pre-pass that builds every layer's
# material with the full transform pipeline and caches it, so the
# lookahead reads consistent data.

## Maximum Skin Angle for Expansion  (`maxSkinAngleForExpansion`)  DONE 2026-05-01
# Cura's setting (default 90°, i.e. effectively off in default profiles).
# When lowered, top/bottom skin expansion is skipped in regions whose
# underlying surface slope exceeds the threshold — protects narrow skin
# slivers near near-vertical walls from being expanded into nothing.
#
# Implemented as: per-layer steep-slope shadow MP (mirror of the existing
# overhang-shadow built for infillOverhangAngle) is built on entry to the
# solid-skin branch; per-region intersection with the shadow toggles the
# allowExpand flag → falls through to skinOverlap-only (no totalExpand).
# isSteepSlopeForSkinExpansion is a pure helper with unit tests covering
# the 90° early-out, 0/45/60/80° thresholds, exact-equality boundary, and
# absolute-slope semantics. Default 90° = no-op; verified in dev preview
# (counterbore re-slice produced identical 199 layers / 17 solid blocks /
# 185k moves / 8.12 MB gcode as the pre-change run).
